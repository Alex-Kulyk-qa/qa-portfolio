## 1. Управление количеством товара (кнопки +/−, поле ввода)

| ID | Проверка | Ожидаемый результат | Статус | Bug ID |
|----|----------|---------------------|--------|--------|
| CL_QTY_001 | Нажатие кнопки `+` | Количество увеличивается на 1, сумма пересчитывается | | |
| CL_QTY_002 | Нажатие кнопки `−` | Количество уменьшается на 1, сумма пересчитывается | | |
| CL_QTY_003 | Нажатие `−` при количестве `1` (граница) | Количество не уходит в 0 или минус; кнопка блокируется или значение остаётся 1 | | |
| CL_QTY_004 | Ввод корректного числа (например, `5`) в поле | Значение принимается, сумма пересчитывается после Update | | |
| CL_QTY_005 | Ввод `0` в поле количества | Система не принимает 0; товар нельзя заказать в нулевом количестве | | |
| CL_QTY_006 | Ввод отрицательного числа (`-3`) | Значение отклоняется или не вводится | | |
| CL_QTY_007 | Ввод букв в поле количества | Поле не принимает буквы либо показывает ошибку | | |
| CL_QTY_008 | Ввод спецсимволов (`@ # $ %`) | Поле не принимает спецсимволы либо показывает ошибку | | |
| CL_QTY_009 | Ввод очень большого числа (`999999`) | Система обрабатывает корректно: принимает или ограничивает разумным пределом, без сбоя | | |
| CL_QTY_010 | Оставить поле количества пустым и нажать Update | Система не принимает пустое значение; показывает ошибку или возвращает прежнее | | |
| CL_QTY_011 | Нажатие кнопки `Update` после изменения количества | Изменения применяются, количество и суммы обновляются | | |
